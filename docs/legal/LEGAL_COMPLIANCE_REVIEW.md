### PIPA 종합 평가

**주요 준수 사항**: 수집 항목 고지, 이용 목적, 보유 기간, 제3자 제공, 위탁처리, 정보주체 권리, 파기, DPO 지정, 안전조치, 동의 기록이 모두 문서화 및 구현됨.

**필수 보완 사항**:
1. HTML 개인정보처리방침에 침해사고 통지 조항 누락 (PIPA 제34조)
2. Markdown 개인정보처리방침의 placeholder 미교체 (제31조)
3. HTTPS 미적용 상태에서 전송 구간 암호화 미충족 (제29조)

| API 키(해시), 메모리 콘텐츠, 프로젝트명 | 서비스 이용 시 (store.py, MemoryStore) | **일치** |

### 4.4 데이터 내보내기 (GDPR Art.20) 실제 작동 여부

**적합** — `export_user_data()` (store.py:1019-1098) 에서 `memory_store` 파라미터를 받아 ChromaDB의 모든 프로젝트 메모리를 포함. dashboard.py:1728-1736에서 registry를 통해 사용자의 MemoryStore를 전달. 내보내기에 포함되는 항목:
- profile (user_id, display_name, email, telegram_id, created_at, tier)
- sessions (created_at, expires_at, last_active, ip_address, user_agent)
- api_keys (label, created_at, last_used)
- consents (document_type, document_ver, consented_at, ip_address)
- memories (프로젝트별: id, content, type, tags, importance, created_at, updated_at)

**이전 리뷰(SECURITY_LEGAL_REVIEW_SUMMARY.md)의 G-2 항목에서 "memory content가 export에 미포함"이라고 기술한 것은 현재 코드와 불일치.** 코드가 업데이트되어 메모리 콘텐츠가 이미 포함되고 있음을 확인.

### 7.1 베타 출시 가능 여부

**조건부 출시 가능 (Conditional GO)**

### 7.4 법적 강점

1. **동의 인프라**: 3중 검증(HTML required, JS check, 백엔드 validation), DB 기록(version, IP, UA) — 업계 모범 사례 수준
2. **데이터 내보내기**: 계정 정보 + 메모리 콘텐츠 포함, JSON 형식, Content-Disposition attachment — GDPR Art.20 충족
3. **계정 삭제 Cascade**: 7개 테이블 순차 삭제 — GDPR Art.17 기본 준수
4. **위탁처리 명시**: Oracle Cloud를 수탁업체로 명시, 위탁 계약 의무 사항 기재
5. **콘텐츠 소유권**: 이용자 소유권 유지 + 최소 이용 허락 + AI 학습/광고 금지 — 이용자 친화적
6. **침해사고 절차서**: 별도 문서로 5단계 절차 수립, 72시간 통지, 증거 보전, 3년 기록 보존
7. **문서 품질**: 한국어로 작성된 12개 조항의 포괄적 개인정보처리방침, 14개 조항의 이용약관 — 베타 서비스 수준을 상회

### 7.5 법적 위험 요약

| 위험 수준 | 항목 | 영향 |
|-----------|------|------|
| **HIGH** | HTTPS 미적용 (F-4) | 전송 구간 암호화 미달 — PIPA 제29조 위반 소지. 단, 초대제 비공개 베타에서는 즉각적인 제재 가능성 낮음 |

**결론**: Memory MCP Server의 법적 문서와 구현은 베타 서비스 출시에 필요한 핵심 요건을 대부분 갖추고 있음. F-1(HTML 침해사고 통지 추가)과 F-2(소재지 기입)는 1-2시간 내 해결 가능하며, F-4(HTTPS)는 인프라 배포가 필요하나 초대제 비공개 베타의 경우 단기적으로 수용 가능. 필수 조치 4건 완료 후 법적으로 안전한 베타 출시가 가능함.
